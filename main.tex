\chapter*{ABSTRACT}
\addcontentsline{toc}{chapter}{ABSTRACT}

Frequent itemset mining is a fundamental task in data mining. Classical algorithms assume binary transaction data where items are either present or absent. However, real-world databases often contain \textbf{uncertainty}: a barcode scanner might be only 80\% confident that it read a product correctly, a sensor might report a reading with known error margins, or data merged from multiple sources might contain conflicting entries. In such databases, each item is associated with an \textbf{existential probability} rather than a simple yes/no flag.

This report addresses the problem of mining \textbf{frequent maximal itemsets} from uncertain transaction databases using two complementary support models. We implement and evaluate four algorithms:

Under the \textbf{expected support model}: \textbf{UGenMax} adapts GenMax (Gouda and Zaki, 2005) using bottom-up weighted tidset intersections, and \textbf{UFPMax} adapts FPMax (Grahne and Zhu, 2003) using top-down conditional database projection. Both employ branch-and-bound search with anti-monotonicity pruning, progressive focusing, and dynamic Top-K threshold raising.

Under the \textbf{probabilistic support model}: \textbf{TODIS-MAX} (based on Sun et al., 2010) provides exact probabilistic frequency computation via Dynamic Programming, and \textbf{APFI-MAX} (based on Chen et al., 2020) provides faster approximate estimation via the Central Limit Theorem.

All algorithms are implemented in pure Java SE with no external dependencies. Experiments on three benchmark datasets (mushroom, retail, accidents) at three uncertainty levels show that: (1) UGenMax and UFPMax produce identical results, with UGenMax being 15--50\% faster; (2) TODIS-MAX and APFI-MAX produce matching results, with APFI-MAX being 3--8$\times$ faster; (3) all algorithms scale to 340K transactions.

\chapter{INTRODUCTION}

\section{What is Frequent Itemset Mining?}

Imagine a supermarket analyzing thousands of customer receipts to discover which products are frequently bought together. If many customers buy bread and butter in the same trip, the pair \{bread, butter\} is a \textbf{frequent itemset}. This information is valuable for product placement, cross-selling, and inventory management.

More formally, given a database of transactions (where each transaction is a set of items) and a minimum support threshold, the goal of frequent itemset mining is to find all item groups that appear in at least that many transactions. This problem was first formalized by Agrawal, Imielinski, and Swami~\cite{agrawal1993} and has since become one of the most studied problems in data mining, with applications in bioinformatics, text mining, web usage analysis, network security, and recommendation systems.

\section{The Challenge of Uncertainty}

Classical algorithms assume perfect data: an item is either in a transaction or it is not. In practice, data is often imperfect. Consider these real-world scenarios:

\begin{itemize}
  \item \textbf{Sensor networks.} A temperature sensor reports values with a known error margin. When the system logs that a sensor detected item A, it might only be 85\% confident in that reading.
  \item \textbf{Data integration.} When merging customer records from two different databases, some matches are ambiguous. A merged record might indicate that a customer purchased item B, but with only 70\% confidence due to name/address mismatches.
  \item \textbf{Privacy preservation.} To protect user privacy, a system deliberately adds random noise to transaction data. Each item's presence is now probabilistic rather than certain.
\end{itemize}

In all these cases, the result is an \textbf{uncertain transaction database} where each item in each transaction carries a probability between 0 and 1. The challenge is: how do we mine meaningful patterns from data that we are not entirely sure about?

\section{Why Maximal Itemsets?}

A database with $m$ distinct items has $2^m$ possible subsets. Even a modest database with 100 items has more possible subsets than atoms in the universe. While most of these are not frequent, the number of frequent itemsets can still be enormous---potentially millions or more.

\textbf{Maximal frequent itemsets} solve this information overload problem. A frequent itemset is maximal if no proper superset of it is also frequent. The key insight is that every frequent itemset is a subset of at least one maximal frequent itemset, so the maximal set is a \textbf{lossless compression} of the full set of frequent itemsets.

\section{Project Objectives}

This project has five objectives:

\begin{enumerate}
  \item Formally define the problem of mining frequent maximal itemsets from uncertain databases using both the expected support model and the probabilistic support model.
  \item Design and implement four algorithms: UGenMax and UFPMax (expected support), TODIS-MAX and APFI-MAX (probabilistic support).
  \item Implement all algorithms as a pure Java SE application with no external library dependencies, with separate algorithm classes and experiment classes.
  \item Experimentally evaluate and compare all four algorithms on benchmark datasets (mushroom, retail, accidents) of varying sizes and uncertainty levels.
  \item Analyze the trade-offs between exact and approximate probabilistic mining.
\end{enumerate}

Chapter~4 presents all four algorithms in detail. Chapter~5 describes the experimental setup. Chapter~6 presents results. Chapter~7 concludes.

\chapter{LITERATURE REVIEW}

\section{Frequent Itemset Mining in Deterministic Databases}

The Apriori algorithm~\cite{agrawal1994} was the first efficient algorithm for frequent itemset mining. Its key contribution was the \textbf{anti-monotonicity property}: if an itemset is infrequent, then all of its supersets are also infrequent. The FP-Growth algorithm~\cite{han2000} improved upon Apriori by eliminating candidate generation through a compressed FP-tree data structure.

\section{Mining Maximal Frequent Itemsets}

Several algorithms have been developed specifically for maximal frequent itemset mining:

\begin{itemize}
  \item \textbf{MaxMiner}~\cite{bayardo1998} was the first to use a look-ahead pruning strategy.
  \item \textbf{GenMax}~\cite{gouda2005} introduced depth-first backtracking with tidsets and progressive focusing. Our UGenMax adapts this algorithm.
  \item \textbf{FPMax}~\cite{grahne2003} extended FP-Growth with MFI-tree superset checking. Our UFPMax adapts this algorithm.
  \item \textbf{CARPENTER-MAX}~\cite{pan2003} was designed for datasets with many items but few transactions.
  \item \textbf{Charm-MFI}~\cite{szathmary2006} discovers maximal itemsets by post-processing closed itemsets.
\end{itemize}

\section{Uncertain Database Mining}

Chui, Kao, and Hung~\cite{chui2007} pioneered uncertain data mining with the expected support model and U-Apriori algorithm. Leung et al.~\cite{leung2008} proposed UF-Growth for uncertain data. Leung and Tanbeer~\cite{leung2013} improved this with UFP-Growth. Li and Zhang~\cite{li2019} addressed the top-k variant with dynamic threshold raising. Aggarwal et al.~\cite{aggarwal2009} analyzed the theoretical complexity of uncertain pattern mining.

\section{Probabilistic Support Mining}

Sun et al.~\cite{sun2010} introduced the \textbf{probabilistic support model}, which defines frequency as $\Pr[\text{sup}(X) \geq T] \geq \tau$, giving a statistically rigorous definition of frequency under uncertainty. Their TODIS framework uses exact PMF computation via Dynamic Programming. Chen et al.~\cite{chen2020} proposed APFI-MAX, which approximates the PMF using the Central Limit Theorem for faster computation, trading small accuracy loss for significant speed improvement ($O(n)$ vs.\ $O(n \cdot T)$).

\section{Gap and Our Contribution}

Most prior work focuses on mining \textbf{all} frequent or \textbf{closed} frequent itemsets from uncertain data. Mining \textbf{maximal} frequent itemsets has received less attention. Furthermore, few works provide a unified comparison across both expected support and probabilistic support models. Our contribution fills these gaps by: (1) adapting GenMax and FPMax for uncertain data with expected support, (2) implementing TODIS-MAX and APFI-MAX for probabilistic maximal mining, and (3) providing comprehensive experimental comparison across all four approaches.

\section{Uncertain Transaction Database}

\begin{definition}[Item Set]
Let $I = \{i_1, i_2, \ldots, i_m\}$ be a finite set of $m$ distinct items.
\end{definition}

\begin{definition}[Uncertain Transaction]
An uncertain transaction $t_k$ is a set of items from $I$, where each item $i$ is associated with an \textbf{existential probability} $P(i, t_k)$ satisfying:
\[ 0 < P(i, t_k) \leq 1 \]
Items are assumed to be mutually independent within each transaction.
\end{definition}

\begin{definition}[Uncertain Transaction Database]
An uncertain transaction database $D = \{t_1, t_2, \ldots, t_n\}$ is an ordered collection of $n$ uncertain transactions.
\end{definition}

\section{Expected Support}

\begin{definition}[Expected Support]
The expected support of an itemset $X = \{x_1, x_2, \ldots, x_p\}$ in database $D$ is:
\begin{equation}
\text{expSup}(X) = \sum_{t \in D} \prod_{x \in X} P(x, t)
\label{eq:expsup}
\end{equation}
\end{definition}

\textbf{Worked Example.} Consider $X = \{1, 3\}$:

\begin{align*}
t_1&: P(1,t_1) \times P(3,t_1) = 0.8 \times 0.6 = 0.48 \\
t_2&: \text{item 1 absent} = 0.00 \\
t_3&: P(1,t_3) \times P(3,t_3) = 0.6 \times 0.7 = 0.42 \\
t_4&: \text{item 1 absent} = 0.00 \\
t_5&: P(1,t_5) \times P(3,t_5) = 0.9 \times 0.8 = 0.72 \\
\end{align*}
\[ \text{expSup}(\{1, 3\}) = 0.48 + 0.00 + 0.42 + 0.00 + 0.72 = \mathbf{1.62} \]

\section{Anti-Monotonicity Property}

\begin{theorem}[Anti-Monotonicity of Expected Support]
For any itemset $X$ and any item $y \notin X$:
\[ \text{expSup}(X \cup \{y\}) \leq \text{expSup}(X) \]
\end{theorem}

\begin{proof}
Since $0 < P(y, t) \leq 1$ for all transactions $t$, each term $\prod_{x \in X} P(x,t) \cdot P(y,t) \leq \prod_{x \in X} P(x,t)$. Therefore the sum cannot increase.
\end{proof}

\section{Frequent and Maximal Frequent Itemsets}

\begin{definition}[Frequent Itemset]
Given threshold minsup, itemset $X$ is \textbf{frequent} if $\text{expSup}(X) \geq \text{minsup}$.
\end{definition}

\begin{definition}[Maximal Frequent Itemset]
A frequent itemset $X$ is \textbf{maximal} if for every item $y \notin X$, $\text{expSup}(X \cup \{y\}) < \text{minsup}$.
\end{definition}

\section{Probabilistic Support}

While expected support provides a single aggregated value, the \textbf{probabilistic support model} captures the full probability distribution of an itemset's support count.

\begin{definition}[Support Random Variable]
Under the Possible World Semantics, the support of itemset $X$ in database $D$ is a random variable:
\begin{equation}
\text{sup}(X) = \sum_{t \in D} \mathbf{1}[X \subseteq t]
\end{equation}
where $\mathbf{1}[X \subseteq t]$ is a Bernoulli random variable with success probability $p_t^X = \prod_{x \in X} P(x, t)$.
Since transactions are independent, $\text{sup}(X)$ follows a \textbf{Poisson Binomial distribution} with parameters $(p_1^X, p_2^X, \ldots, p_n^X)$.
\end{definition}

\begin{definition}[Probabilistic Frequent Itemset (p-FP)]
Given integer threshold $T$ (minsup) and probability threshold $\tau$ (minprob), itemset $X$ is a \textbf{probabilistic frequent itemset} (p-FP) if:
\begin{equation}
\Pr[\text{sup}(X) \geq T] \geq \tau
\end{equation}
\end{definition}

\begin{definition}[Maximal Probabilistic Frequent Itemset]
A p-FP $X$ is \textbf{maximal} if no proper superset of $X$ is also a p-FP.
\end{definition}

The expected support relates to the probabilistic support: $E[\text{sup}(X)] = \sum_t p_t^X = \text{expSup}(X)$.
The variance is: $\text{Var}[\text{sup}(X)] = \sum_t p_t^X (1 - p_t^X)$.

\section{Problem Statement}

This project addresses two variants of the problem:

\textbf{Problem 1 (Expected Support Model):}
\begin{itemize}
  \item \textbf{Input:} Uncertain database $D$, threshold $\text{minsup} > 0$ (or integer $K$ for Top-K mode).
  \item \textbf{Output:} All maximal frequent itemsets where $\text{expSup}(X) \geq \text{minsup}$.
  \item \textbf{Algorithms:} UGenMax, UFPMax.
\end{itemize}

\textbf{Problem 2 (Probabilistic Support Model):}
\begin{itemize}
  \item \textbf{Input:} Uncertain database $D$, integer threshold $T$, probability threshold $\tau \in (0, 1]$.
  \item \textbf{Output:} All maximal p-FPs where $\Pr[\text{sup}(X) \geq T] \geq \tau$.
  \item \textbf{Algorithms:} TODIS-MAX (exact), APFI-MAX (approximate).
\end{itemize}

\chapter{METHODS}

This chapter presents both algorithms in detail: data structures, pseudocode, worked examples, and complexity analysis.

\section{Branch-and-Bound Framework}

Both algorithms explore the \textbf{itemset lattice} using depth-first search (DFS). The search is a tree: the root is the empty set, each node is an itemset (prefix), and each edge adds one item. Without pruning, this requires exploring $2^m$ subsets. Branch-and-bound makes the search tractable by computing \textbf{bounds} at each node and \textbf{pruning} subtrees that cannot yield useful results.

\subsection{Pruning 1: Anti-Monotonicity Bound}

\textbf{Rule:} If $\text{expSup}(\text{prefix} \cup \{\text{item}\}) < \text{minsup}$, skip this item and the entire subtree below it.

\textbf{Rationale:} By Theorem~3.1, every superset will have even lower expected support. This rule is responsible for the majority of pruning in practice. For UGenMax, a cheaper upper bound $\min(\text{expSup}(A), \text{expSup}(B))$ is checked before the full intersection.

\subsection{Pruning 2: Progressive Focusing}

\textbf{Rule:} Let $P$ be the current prefix and $C$ the remaining candidates. If $P \cup C \subseteq M$ for some known MFI $M$, prune the entire subtree.

\textbf{Rationale:} Any discoverable itemset would be a subset of $M$ and therefore not maximal. This rule becomes increasingly effective as more MFIs are discovered during the search.

\subsection{Pruning 3: Top-K Dynamic Threshold Raising}

In Top-K mode, a \textbf{min-heap} of size $K$ tracks the best results. When a new MFI is discovered, if the heap is full and the new MFI's expected support exceeds the heap minimum, the minimum is replaced. The new heap minimum becomes the updated minsup.

\textbf{Feedback loop:} Better results $\rightarrow$ higher threshold $\rightarrow$ more aggressive pruning $\rightarrow$ faster convergence.

\textbf{Design difference:} UGenMax raises the threshold dynamically during search. UFPMax does \textbf{not}, because its processing order discovers single-item MFIs early, which would prematurely block multi-item MFIs.

\section{Algorithm 1: UGenMax}

UGenMax adapts GenMax~\cite{gouda2005} for uncertain data using \textbf{bottom-up, depth-first backtracking} with \textbf{weighted tidsets}.

\subsection{Data Structure: Weighted Tidsets}

\begin{definition}[Weighted Tidset]
The weighted tidset of item $i$ is:
\[ \text{WT}(i) = \{(t, P(i,t)) \mid t \in D,\ P(i,t) > 0\} \]
sorted by transaction ID. Expected support: $\text{expSup}(i) = \sum_{(t,p) \in \text{WT}(i)} p$.
\end{definition}

To compute $\text{expSup}(\{A, B\})$, we \textbf{intersect} $\text{WT}(A)$ and $\text{WT}(B)$ using a merge-join in $O(|\text{WT}(A)| + |\text{WT}(B)|)$ time. For each common transaction $t$, the result is $(t, P(A,t) \times P(B,t))$.

\textbf{Worked Example.} Using items 1 and 3 from Table~\ref{tab:running_example}:

\begin{align*}
\text{WT}(1) &= [(t_1, 0.8),\ (t_3, 0.6),\ (t_5, 0.9)] \quad \text{expSup} = 2.3 \\
\text{WT}(3) &= [(t_1, 0.6),\ (t_2, 0.5),\ (t_3, 0.7),\ (t_5, 0.8)] \quad \text{expSup} = 2.6
\end{align*}

Intersection:
\begin{align*}
t_1&: \text{both present} \rightarrow 0.8 \times 0.6 = 0.48 \\
t_2&: \text{only in WT}(3) \rightarrow \text{skip} \\
t_3&: \text{both present} \rightarrow 0.6 \times 0.7 = 0.42 \\
t_5&: \text{both present} \rightarrow 0.9 \times 0.8 = 0.72
\end{align*}
\[ \text{WT}(\{1,3\}) = [(t_1, 0.48),\ (t_3, 0.42),\ (t_5, 0.72)] \quad \text{expSup} = 1.62 \]

\subsection{Enumeration Strategy}

Items are sorted by expected support \textbf{ascending}. Candidates are processed \textbf{most frequent first} (right to left). This ordering discovers large MFIs early, strengthening progressive focusing.

\subsection{Pseudocode}

\begin{algorithm}[H]
\caption{UGenMax}
\begin{algorithmic}[1]
\Require Uncertain database $D$, threshold minsup
\Ensure Set $\text{MFI}$ of all maximal frequent itemsets
\State $\text{MFI} \gets \emptyset$
\For{each item $i$ in $D$}
  \State Build $\text{WT}(i)$ by scanning all transactions
\EndFor
\State Remove items where $\text{expSup}(i) < \text{minsup}$
\State Sort items by expSup ascending: $[i_1, i_2, \ldots, i_k]$
\State \Call{GenMax}{$\emptyset$, $[i_1, \ldots, i_k]$, null}
\State \Return MFI
\end{algorithmic}
\end{algorithm}

\begin{algorithm}[H]
\caption{GenMax Procedure}
\begin{algorithmic}[1]
\Procedure{GenMax}{prefix, candidates, prefixTidset}
  \If{$\text{prefix} \cup \text{candidates} \subseteq$ some MFI in MFI} \Comment{Pruning 2}
    \State \Return
  \EndIf
  \State hasFreqExt $\gets$ false
  \For{idx $\gets$ $|$candidates$| - 1$ \textbf{downto} 0}
    \State item $\gets$ candidates[idx]
    \State newTidset $\gets$ \Call{Intersect}{prefixTidset, WT(item)}
    \State expSup $\gets$ Sum(newTidset.probabilities)
    \If{expSup $<$ minsup} \Comment{Pruning 1}
      \State \textbf{continue}
    \EndIf
    \State hasFreqExt $\gets$ true
    \State newPrefix $\gets$ prefix $\cup$ \{item\}
    \State newCands $\gets$ \{$c \in$ candidates$[0..idx{-}1]$ $|$ Intersect(newTidset, WT($c$)).expSup $\geq$ minsup\}
    \If{newCands $= \emptyset$}
      \State \Call{AddMaximal}{newPrefix, expSup}
    \Else
      \State \Call{GenMax}{newPrefix, newCands, newTidset}
    \EndIf
  \EndFor
  \If{\textbf{not} hasFreqExt \textbf{and} prefix $\neq \emptyset$}
    \State \Call{AddMaximal}{prefix, prefixTidset.expSup}
  \EndIf
\EndProcedure
\end{algorithmic}
\end{algorithm}

\subsection{Complexity Analysis}

\textbf{Time:} Worst case $O(2^m)$ nodes, each with $O(n)$ tidset intersection. Pruning dramatically reduces explored nodes in practice.

\textbf{Space:} $O(m \times n)$ for weighted tidsets, $O(|\text{MFI}|)$ for results, $O(m)$ for the recursion stack.

\section{Algorithm 2: UFPMax}

UFPMax adapts FPMax~\cite{grahne2003} for uncertain data using \textbf{top-down, recursive conditional database projection}.

\subsection{Data Structure: Projected Transactions}

Each projected transaction is a triple (weight, items[], probs[]):
\begin{itemize}
  \item \textbf{weight}: accumulated product of probabilities of all prefix items. Initially 1.0.
  \item \textbf{items[] and probs[]}: remaining items and probabilities, sorted by frequency-descending order.
\end{itemize}

Expected support of item $i$ in a conditional database:
\begin{equation}
\text{expSup}(i \mid \text{prefix}) = \sum_{t : i \in t} \text{weight}_t \times P(i, t)
\end{equation}

\subsection{Conditional Database Construction}

When extending prefix by item $i$:
\begin{enumerate}
  \item Select transactions containing item $i$.
  \item Compute $\text{new\_weight} = \text{old\_weight} \times P(i, t)$.
  \item Project to items \textbf{before} $i$ in header order (more frequent items).
  \item Remove items with conditional expSup $<$ minsup.
\end{enumerate}

\subsection{Pseudocode}

\begin{algorithm}[H]
\caption{UFPMax}
\begin{algorithmic}[1]
\Require Uncertain database $D$, threshold minsup
\Ensure Set $\text{MFI}$ of all maximal frequent itemsets
\State $\text{MFI} \gets \emptyset$
\State Compute expSup for each single item
\State Remove items with expSup $<$ minsup
\State Sort items by expSup descending: header $= [h_1, \ldots, h_k]$
\State Build initial projected DB (weight = 1.0, frequent items only)
\State \Call{FPMax}{$\emptyset$, 0.0, projDB, header}
\State \Return MFI
\end{algorithmic}
\end{algorithm}

\begin{algorithm}[H]
\caption{FPMax Procedure}
\begin{algorithmic}[1]
\Procedure{FPMax}{prefix, prefixExpSup, condDB, headerItems}
  \If{prefix $\cup$ headerItems $\subseteq$ some MFI} \Comment{Pruning 2}
    \State \Return
  \EndIf
  \State hasFreqExt $\gets$ false
  \For{idx $\gets$ $|$headerItems$| - 1$ \textbf{downto} 0}
    \State item $\gets$ headerItems[idx]
    \State expSup $\gets \sum_{t \in \text{condDB}}$ weight$_t \times P$(item, $t$)
    \If{expSup $<$ minsup} \Comment{Pruning 1}
      \State \textbf{continue}
    \EndIf
    \State hasFreqExt $\gets$ true; newPrefix $\gets$ prefix $\cup$ \{item\}
    \State newCondDB $\gets$ \Call{BuildConditional}{condDB, item, idx}
    \State newHeader $\gets$ \{$h \in$ header$[0..idx{-}1]$ $|$ condExpSup$(h) \geq$ minsup\}
    \If{newHeader $= \emptyset$}
      \State \Call{AddMaximal}{newPrefix, expSup}
    \Else
      \State \Call{FPMax}{newPrefix, expSup, newCondDB, newHeader}
    \EndIf
  \EndFor
  \If{\textbf{not} hasFreqExt \textbf{and} prefix $\neq \emptyset$}
    \State \Call{AddMaximal}{prefix, prefixExpSup}
  \EndIf
\EndProcedure
\end{algorithmic}
\end{algorithm}

\subsection{Complexity Analysis}

\textbf{Time:} Worst case $O(2^m)$ nodes. Conditional DB construction is $O(|\text{condDB}| \times |\text{items}|)$ per node, shrinking at deeper levels.

\textbf{Space:} $O(n \times m)$ for the initial projected DB. Each recursive level maintains its own conditional DB. Stack depth at most $m$.

\section{Algorithm 3: TODIS-MAX}

TODIS-MAX (Top-Down Inheritance of Support for Maximal itemsets) uses the \textbf{probabilistic support model} to mine maximal p-FPs with exact probability computation. It is based on the framework of Sun et al.~\cite{sun2010}.

\subsection{Probabilistic Support Computation}

The key challenge is computing $\Pr[\text{sup}(X) \geq T]$ for a given itemset $X$. Since $\text{sup}(X)$ follows a Poisson Binomial distribution, the probability mass function (PMF) can be computed via \textbf{Dynamic Programming}:

Let $f_k(j)$ = probability that $j$ of the first $k$ transactions contain $X$. Then:
\begin{align}
f_0(0) &= 1 \\
f_k(j) &= f_{k-1}(j) \cdot (1 - p_k^X) + f_{k-1}(j-1) \cdot p_k^X
\end{align}

This requires $O(n \cdot T)$ time and $O(T)$ space. Then $\Pr[\text{sup}(X) \geq T] = \sum_{j=T}^{n} f_n(j)$.

\subsection{Algorithm Overview}

\textbf{Phase 1 --- Apriori-based Candidate Generation:}
\begin{enumerate}
  \item Start with single items; apply level-wise join and pruning.
  \item \textbf{Pruning by count:} If $\text{cnt}(X) < T$, prune (Lemma 2 from~\cite{sun2010}). Here $\text{cnt}(X)$ is the number of transactions containing all items in $X$.
  \item \textbf{Pruning by Chernoff bound:} If $E[\text{sup}(X)]$ is too low for $\Pr[\text{sup}(X) \geq T] \geq \tau$ to hold, prune without computing the full PMF.
\end{enumerate}

\textbf{Phase 2 --- Top-Down Maximal Verification:}
\begin{enumerate}
  \item Sort candidates by length descending.
  \item For each candidate (longest first), skip if it is a subset of any confirmed maximal p-FP.
  \item Compute exact PMF via DP and check $\Pr[\text{sup}(X) \geq T] \geq \tau$.
  \item Confirmed candidates become maximal p-FPs.
\end{enumerate}

\subsection{Pseudocode}

\begin{algorithm}[H]
\caption{TODIS-MAX}
\begin{algorithmic}[1]
\Require Uncertain database $D$, integer threshold $T$, probability threshold $\tau$
\Ensure Set of all maximal p-FPs
\State $C_1 \gets$ single items passing count and Chernoff pruning
\State $k \gets 1$
\While{$C_k \neq \emptyset$}
  \State $C_{k+1} \gets$ Apriori-join($C_k$) with count + Chernoff pruning
  \State $k \gets k + 1$
\EndWhile
\State Sort all candidates by length descending
\State $\text{MFI} \gets \emptyset$
\For{each candidate $X$ (longest first)}
  \If{$X \subseteq$ some $M \in \text{MFI}$}
    \State \textbf{skip} \Comment{Subset of known maximal}
  \EndIf
  \State Compute PMF via DP, get $\Pr[\text{sup}(X) \geq T]$
  \If{$\Pr[\text{sup}(X) \geq T] \geq \tau$}
    \State $\text{MFI} \gets \text{MFI} \cup \{X\}$
  \EndIf
\EndFor
\State \Return MFI
\end{algorithmic}
\end{algorithm}

\subsection{Complexity Analysis}

\textbf{Time:} Candidate generation is $O(\sum |C_k|^2)$. PMF computation is $O(n \cdot T)$ per candidate. Phase 2 processes at most $\sum |C_k|$ candidates.

\textbf{Space:} $O(T)$ for the DP array, $O(\sum |C_k|)$ for storing candidates.

\section{Algorithm 4: APFI-MAX}

APFI-MAX (Approximation of Probabilistic Frequent Itemset-MAX) is a faster alternative to TODIS-MAX that uses the \textbf{Central Limit Theorem (CLT)} for approximate probability estimation. It is based on Chen et al.~\cite{chen2020}.

\subsection{CLT Approximation}

For large $n$, the Poisson Binomial $\text{sup}(X)$ is approximately Normal:
\begin{equation}
\text{sup}(X) \sim \mathcal{N}\left(\mu, \sigma^2\right), \quad \mu = \sum_t p_t^X, \quad \sigma^2 = \sum_t p_t^X(1 - p_t^X)
\end{equation}

The frequency is estimated as:
\begin{equation}
\Pr[\text{sup}(X) \geq T] \approx 1 - \Phi\left(\frac{T - \mu}{\sigma}\right)
\label{eq:clt}
\end{equation}
where $\Phi$ is the standard normal CDF. This runs in $O(n)$ per itemset --- significantly faster than the $O(n \cdot T)$ exact DP.

\subsection{Expectation Bounds}

APFI-MAX uses tighter pruning via expectation bounds derived from the CLT:

\begin{itemize}
  \item \textbf{Lower bound:} If $E[\text{sup}(X)] < lb(E)$, then $X$ cannot be a p-FP regardless of variance. The lower bound $lb(E)$ is computed from the Chernoff inequality.
  \item \textbf{Upper bound:} If $E[\text{sup}(X)] \geq ub(E)$, then $X$ is definitely a p-FP. The upper bound is $ub(E) = T + z_\tau \cdot \sqrt{E}$, where $z_\tau = \Phi^{-1}(1 - \tau)$.
\end{itemize}

\subsection{Algorithm Overview}

\textbf{Step 1 --- CGEB (Candidate Generation with Expectation Bound):}
Apriori-based generation with two criteria per candidate:
(a) $\text{cnt}(X) \geq T$, and (b) $E[\text{sup}(X)] \geq lb(E)$.

\textbf{Step 2 --- Top-Down Confirmation:}
Process candidates from longest to shortest. If a candidate is a subset of a confirmed p-FP from a longer level, it inherits frequent status (skip the check). Otherwise, apply FM.

\textbf{Step 3 --- FM (Frequency Measurement):}
If $E[\text{sup}(X)] \geq ub(E)$, automatically frequent. Otherwise, estimate via CLT (Equation~\ref{eq:clt}).

\subsection{Pseudocode}

\begin{algorithm}[H]
\caption{APFI-MAX}
\begin{algorithmic}[1]
\Require Uncertain database $D$, integer threshold $T$, probability threshold $\tau$
\Ensure Set of maximal p-FPs (approximate)
\State Compute $lb(E)$ and $ub(E)$ from $T$ and $\tau$
\State $C_1 \gets$ items with cnt $\geq T$ and $E \geq lb(E)$
\State Generate higher-level candidates via Apriori with expectation bound
\State Sort all candidates by length descending
\State $\text{MFI} \gets \emptyset$; $\text{Fre\_Pre} \gets \emptyset$
\For{each candidate $X$ (longest first)}
  \If{$X \subseteq$ some $Y \in \text{Fre\_Pre}$}
    \State Mark $X$ as frequent (inherited) \Comment{Skip verification}
    \State \textbf{continue}
  \EndIf
  \If{$E(X) \geq ub(E)$}
    \State Mark $X$ as frequent \Comment{Upper bound}
  \ElsIf{$1 - \Phi\left(\frac{T - E(X)}{\sqrt{\text{Var}(X)}}\right) \geq \tau$}
    \State Mark $X$ as frequent \Comment{CLT}
  \EndIf
  \If{$X$ is frequent and $X \not\subseteq$ any $M \in \text{MFI}$}
    \State $\text{MFI} \gets \text{MFI} \cup \{X\}$
  \EndIf
\EndFor
\State \Return MFI
\end{algorithmic}
\end{algorithm}

\subsection{Complexity Analysis}

\textbf{Time:} Same as TODIS-MAX for candidate generation. Frequency measurement is $O(n)$ per candidate (vs. $O(n \cdot T)$ for exact DP), giving a significant speedup on large datasets.

\textbf{Space:} $O(\sum |C_k|)$ for candidates plus $O(1)$ per frequency check (no DP array needed).

\section{Top-K Seeding}

Both UGenMax and UFPMax support a Top-K seeding phase: (1) take the top-50 most frequent items, (2) generate all single and 2-item combinations, (3) fill a Top-K heap, (4) set initial minsup to 50\% of the heap minimum, (5) reset the heap and begin the main search. If fewer than $K$ results are found, retry with 25\% of current threshold, up to 3 retries.

\section{Maximality Checking}

When a leaf node is reached, the candidate MFI is checked: (1) it must not be a subset of any existing MFI, and (2) existing MFIs that are subsets of the new candidate are removed. Both checks use sorted arrays for $O(|A| + |B|)$ merge-based comparison.

\section{Cross-Model Comparison}

The expected support model (UFPMax/UGenMax) and probabilistic support model (TODIS-MAX/APFI-MAX) serve complementary purposes:

\begin{itemize}
  \item \textbf{Expected support} is faster and simpler but treats all transactions equally: an itemset with 0.01 probability in each of 1000 transactions can have the same expected support as one with 1.0 probability in 10 transactions.
  \item \textbf{Probabilistic support} distinguishes these cases by asking ``what is the probability that this itemset appears at least $T$ times?'' This gives a more statistically rigorous measure of frequency.
  \item For practitioners: use expected support for quick exploratory analysis, and probabilistic support when statistical guarantees matter.
\end{itemize}

\chapter{CONCLUSIONS}

\section{Summary}

This project designed, implemented, and evaluated four algorithms for mining frequent maximal itemsets from uncertain transaction databases, covering two support models:

\begin{itemize}
  \item \textbf{Expected support model:} UGenMax (bottom-up weighted tidset intersections, adapted from GenMax) and UFPMax (top-down conditional database projection, adapted from FPMax). Both use branch-and-bound with three pruning strategies: anti-monotonicity, progressive focusing, and dynamic Top-K threshold raising.
  \item \textbf{Probabilistic support model:} TODIS-MAX (exact PMF computation via Dynamic Programming, based on Sun et al., 2010) and APFI-MAX (approximate frequency estimation via CLT, based on Chen et al., 2020). Both use Apriori-based candidate generation with top-down maximal verification.
\end{itemize}

Experiments on three benchmark datasets (mushroom, retail, accidents) at three uncertainty levels demonstrate that all algorithms produce correct results and exhibit distinct performance characteristics.

\section{Key Contributions}

\begin{enumerate}
  \item Formalized the problem with precise mathematical definitions for both expected support and probabilistic support models.
  \item Adapted GenMax and FPMax for uncertain data with expected support, including Top-K mode.
  \item Implemented TODIS-MAX and APFI-MAX for probabilistic maximal frequent itemset mining.
  \item Clean Java SE implementation with no external dependencies, organized into algorithm classes (no main) and experiment classes (with main and statistics output).
  \item Comprehensive experimental comparison across 3 datasets, 3 uncertainty levels, 4 algorithms, and varying parameter configurations, revealing that UGenMax outperforms UFPMax and APFI-MAX outperforms TODIS-MAX in speed.
\end{enumerate}

\section{Limitations}

\begin{itemize}
  \item Maximality checking uses linear scan ($O(|\text{MFI}|)$ per check); a trie or hash-based approach could improve this.
  \item APFI-MAX's CLT approximation may be less accurate for small datasets where the normal approximation is poor.
  \item TODIS-MAX and APFI-MAX do not support Top-K mode.
  \item Memory usage on the accidents dataset reaches 2 GB at low thresholds.
\end{itemize}

\section{Future Work}

Possible extensions include:
\begin{itemize}
  \item Diffset optimization for UGenMax to reduce memory usage.
  \item Parallel search across independent subtrees for multi-core speedup.
  \item Extension to mining closed or sequential patterns from uncertain data.
  \item Evaluation on real-world uncertain datasets from sensor networks or noisy data integration.
  \item Top-K mode for probabilistic algorithms.
  \item GPU acceleration for the DP-based PMF computation in TODIS-MAX.
\end{itemize}
